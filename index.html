¿Qué es el DOM?

Las siglas DOM significan Document Object Model, o lo que es lo mismo, la estructura de un documento HTML. Una página HTML está formada por múltiples etiquetas HTML, anidadas una dentro de otra, formando un árbol de etiquetas relacionadas entre sí, que se denomina árbol DOM (o simplemente DOM).

Observa que como la etiqueta <head> está dentro de <html> se dibuja como hija de <html>. La etiqueta <title> está dentro de <head>, por lo que será hija de esta y así sucesivamente.

En Javascript, cuando nos referimos al DOM nos referimos a esta estructura de árbol, mediante la cuál podemos acceder a ella y modificar los elementos del HTML desde Javascript, añadiendo nuevas etiquetas, modificando o eliminando otras, cambiando sus atributos HTML, añadiendo clases, cambiando el contenido de texto, etc...


Principales Características

1. Estructura

  • El DOM representa el documento como un árbol de nodos.
  • Cada elemento HTML es un nodo, y los nodos pueden contener otros nodos como hijos.


2. Accesible y manipulable con JavaScript

  • Puedes acceder, modificar, agregar o eliminar elementos HTML desde JS.

3. Representa todo el documento

  • No solo etiquetas: también incluye atributos, texto, comentarios, etc., como nodos.

4. Interfaz orientada a objetos

  • Cada nodo es un objeto con propiedades y métodos.
  • Ejemplo: elemento.classList, elemento.appendChild(), elemento.remove().

5. Dinámico y en tiempo real

  • Los cambios en el DOM se reflejan inmediatamente en la página sin necesidad de recargarla.

6. Lenguaje independiente

  • El DOM puede ser manipulado desde cualquier lenguaje de programación que soporte su API, aunque JavaScript es el más común en navegadores.

Objetivos y utilidad del DOM en desarrollo web


  1. Representar la estructura del documento: El DOM convierte el HTML en una estructura en forma de árbol que puede ser accedida y manipulada.
  2. Permitir la manipulación dinámica de contenido: Permite cambiar, añadir o eliminar elementos y contenido en la página sin recargarla.
  3. Facilitar la interacción con el usuario: A través de eventos, el DOM ayuda a responder a acciones como clics, teclas presionadas, formularios, etc.
  4. Integración con lenguajes de programación: Principalmente con JavaScript, para crear páginas web interactivas y dinámicas.
  5. Soportar la actualización en tiempo real: Cambios realizados en el DOM se reflejan inmediatamente en la interfaz de usuario.
  6. Estandarización: Proporciona una API estándar para acceder y manipular documentos HTML y XML en cualquier navegador.
